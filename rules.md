# RCV Corporation – Website Rules & Guidelines

## 1. Language

- All visible content MUST be in English
- Commands and prompts can be in Portuguese
- Never mix languages in the UI
- Use a translation system (data-i18n + translations.js)

---

## 2. Design Style

- Corporate, modern, and professional
- Inspired by infrastructure, engineering, and energy companies
- Clean layout with strong hierarchy
- Use large sections with breathing space
- Avoid clutter

---

## 3. Color System

Use a consistent color palette:

Primary:
- Dark Blue: #0B1F3A

Secondary:
- Blue Accent: #1E3A8A

Neutral:
- White: #FFFFFF
- Light Gray: #F5F7FA
- Medium Gray: #6B7280
- Dark Gray: #111827

Usage:
- Background: white or light gray
- Text: dark gray or dark blue
- Buttons: primary blue
- Hover states: slightly lighter blue

---

## 4. Typography

- Use clean, modern sans-serif fonts
- Example: Inter, Open Sans, or similar

Hierarchy:
- H1: 40–56px
- H2: 28–36px
- H3: 20–24px
- Body: 16–18px

Rules:
- Strong contrast
- Clear hierarchy
- Avoid too many font styles

---

## 5. Spacing System

Use a consistent spacing scale (8px system):

- 8px
- 16px
- 24px
- 32px
- 48px
- 64px
- 80px

Rules:
- Sections must have large vertical spacing (at least 64px–80px)
- Keep consistent padding inside containers
- Avoid random spacing values

---

## 6. Layout

- Use a max-width container (1200px–1280px)
- Center content horizontally
- Use grid or flexbox
- Maintain consistent alignment across pages

---

## 7. Components

### Header
- Same across all pages
- Sticky or fixed
- Clear navigation

### Footer
- Same across all pages
- Include company name and links

### Buttons
- Primary: solid blue
- Secondary: outline or subtle
- Consistent padding and border-radius

### Sections
Each section should include:
- Title
- Description
- Visual (image or icon)

---

## 8. Images

- Use high-quality placeholder images
- Focus on:
  - construction
  - infrastructure
  - heavy equipment
  - engineering environments

- Avoid generic stock photos (people smiling at laptop)

---

## 9. Navigation

- Same navigation on all pages
- Active page must be highlighted
- Keep it simple:
  - Home
  - About
  - Services
  - ESG
  - Contact

---

## 10. Responsiveness

- Must work on:
  - desktop
  - tablet
  - mobile

Rules:
- Stack content on mobile
- Maintain spacing and readability
- Avoid horizontal scroll

---

## 11. Code Standards

- Use semantic HTML
- Keep code clean and readable
- Avoid inline styles when possible
- Reuse classes
- Keep CSS organized

---

## 12. Consistency

- Use the same spacing, colors, and typography across all pages
- Reuse components whenever possible
- Maintain visual and structural consistency

---

## 13. Content Structure

Each page should follow a clear structure:

- Hero section
- Content sections
- Call-to-action
- Footer

---

## 14. Call to Action

- Always include a CTA section
- Primary CTA: "Contact Us"
- Clear and visible buttons

---

## 15. Performance (Basic)

- Optimize images
- Avoid unnecessary scripts
- Keep the website lightweight

---

## 16. Goal

The website should feel:

- Professional
- Trustworthy
- Corporate-level quality
- Suitable for partners and investors
